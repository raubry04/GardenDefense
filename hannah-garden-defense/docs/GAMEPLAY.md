# Gameplay Reference

*Audience: designers and developers tuning the game*

Canonical numbers live in [`src/config.js`](../src/config.js). This document describes player-facing behavior.

## Towers

| Key | Role | Notes |
|-----|------|-------|
| RABBIT | AoE slow | Strong early control |
| CHICKEN | Single-target damage | Multi-egg at higher tiers; no vs Parrot fly / Elephant armor |
| DOG | AoE stun + slow | Good vs fast ground units |
| OWL | Long-range sniper | Targets enemy closest to gate |
| DUCK | AoE slow | Stronger slow than Rabbit |
| PENGUIN | AoE freeze pulse | Cooldown-based |
| PIG_WALL | Path blocker | Enemies must destroy it; thorns at max tier |

Unlock rules: `level` (Hannah XP) or `zone` index in `GameConfig.towers.*.unlock`.

Meta upgrades (between battles) apply tier stats via `tower_upgrades` progress — see [Balance Guide](BALANCE.md).

## Enemies

| Key | Role | Special |
|-----|------|---------|
| SNAKE | Basic | — |
| FROG | Splitter | Spawns Snakes on death |
| GORILLA | Fast | Immune to slow |
| PARROT | Flyer | Ignores path and Pig Walls |
| MONKEY | Thief | May steal sunshine when hit |
| BEAR | Tower hunter | Leaves path to attack towers |
| ELEPHANT | Boss | Armored vs Chicken; stomp slows towers |
| COW | Tank | High HP, slow (zone 2+) |
| HORSE | Runner | Very fast + speed bonus (zone 3+) |
| BUFFALO | Mini-boss | Stun/freeze immune; extra Pig Wall damage (zone 4+) |

First appearance in a battle shows a short toast from `GameConfig.enemyIntros`.

## Zones

Five story zones plus endless (zone index ≥ `zones.length`):

1. Sunflower Meadow — Snake, Frog  
2. Vegetable Garden — + Gorilla, Parrot, Cow  
3. Chicken Coop — + Monkey, Horse  
4. Berry Patch — + Bear, Buffalo  
5. Apple Orchard — + Elephant, Buffalo (boss battle uses last pool entry)

Each zone has 5 battles; the final battle is a **boss battle** (+ extra waves, elite spawns in last third).

## Waves

Generated by `WaveManager` using `GameConfig.waves`:

- Wave count scales by zone (`minWaves` → `maxWaves`).
- Per-wave enemy count uses `baseCount`, `countPerWave`, `countPerBattle`, and boss extras.
- Zone 0 battle 0: gentler waves 1–2 (mostly Snake, lower counts) via `zoneIntro`.
- Preview API: `getNextWavePreview()` — used by HUD between waves.

## Economy (in-battle)

- Starting sunshine per zone: `startingSunshinePoints`.
- Kill rewards: per-enemy `reward`.
- Wave complete: `waveCompletionBonus`.
- Early send: `earlyWaveBonusPoints`.
- Sell refund: `sellRefundPercent` of placement cost.

## Lives & leaks

- `startingLives` hearts; each enemy reaching the gate drains `min(enemy.damage, maxLifeLossPerLeak)` lives.
- `maxLifeLossPerLeak` is an anti-frustration cap (default `5`) so a single big enemy can't wipe half the life bar in one leak.
- Every leak shows an "An animal got in! -N" callout at the gate (`EnemyBehavior._showGateLeakFeedback`).

## Stars & meta

- Star thresholds: `starThresholds` (lives remaining).
- Bank rate on victory: `metaSunshineBankRate` of battle sunshine earned.
- The upgrade bank is tracked as monotonic `metaSunshineEarned` / `metaSunshineSpent` totals; the spendable balance is `available = max(0, earned - spent)` (`availableMetaBank`). This keeps sync merge-safe (both totals MAX-merge) while spending still persists across devices. See [BALANCE.md](BALANCE.md#meta-sunshine-bank-earned-vs-spent).
- Hannah XP from kills and battle completion; levels unlock towers/abilities.
- Leveling up mid-battle fires a "⭐ Level Up!" toast, calling out any tower/ability/passive unlocked at that level (`levelUpUnlockMessage` in `hannahProgress.js`).

## Abilities
