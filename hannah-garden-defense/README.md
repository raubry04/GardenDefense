# Hannah's Garden Defense

A mobile-first tower defense game for kids. Hannah and her animal friends defend the garden against waves of critters along winding paths.

Built with **Phaser 3**, **Vite**, and a small **Express + SQLite** backend for progress and leaderboards.

## Play (family setup)

1. Install **Node.js 20 or newer** from [nodejs.org](https://nodejs.org).
2. Ensure source art packs sit **one folder above** this project:
   - `../Assets/` (Craftpix ground tiles, props, UI)
   - `../Animal Pack Remastered/` (animal sprites)
3. Open a terminal in this folder (`hannah-garden-defense/`).
4. First-time setup:

   ```bash
   npm run setup
   ```

   This installs dependencies, copies the 78 runtime asset files into `assets/`, and builds the production bundle.

5. Start the game:

   ```bash
   start.bat
   ```

   Or: `npm run full-start`

6. **This PC:** http://localhost:5050  
   **Phone/tablet (same Wi‑Fi):** `http://<your-pc-ip>:5050` — find the IP with `ipconfig` (IPv4 on your Wi‑Fi adapter).

Hard-refresh after updates (Ctrl+Shift+R / clear Safari cache on iOS).

## How to play

- Tap a tower card, then tap a grass tile to place a defender.
- Enemies follow the dirt path — stop them before they reach the garden gate.
- Earn Sunshine Points from kills and completed waves; upgrade towers and unlock new defenders as Hannah levels up.
- Use Hannah's ability buttons on the right; pause via the HUD button or Escape.

```

├── assets/              Runtime game files (generated by npm run assets)
├── dist/                Production build (generated by npm run build)
├── docs/                Developer documentation
├── scripts/             Asset collection & build helpers

```

## npm scripts

| Script | Purpose |
|--------|---------|
| `npm run setup` | `npm install` + collect assets + production build |
| `npm run dev` | Vite dev server (port 5173, proxies `/api` to 5050) |
| `npm run dev:all` | Vite + Express together (full stack with HMR) |
| `npm run build` | Production bundle → `dist/` + copy assets → `dist/game-assets/` |
| `npm run start` | Run Express server on port 5050 |
| `npm run assets` | Copy used files from parent packs into `assets/` |
| `npm run test` | Run Vitest unit tests |
| `npm run validate` | Tests + runtime asset manifest check |
| `npm run full-start` | Build then start server |

## Tech stack

| Layer | Choice |
|-------|--------|
| Game engine | Phaser 3 |
| Frontend | ES modules, HTML5 canvas |
| Build | Vite 5 |
| Backend | Node.js, Express |
| Database | SQLite (`better-sqlite3`) |
| Map tiles | Craftpix (via `craftpixTiles.js`) |
| Characters | Animal Pack Remastered |

## Asset URLs

- **JS bundle:** `/assets/index-*.js` (Vite output)
- **Game media:** `/game-assets/...` (images, audio, fonts)

Do not copy game media into `dist/assets/` — that path is reserved for the JS bundle.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| Green screen / MIME type error | Rebuild (`npm run build`) and hard-refresh; JS must load from `/assets/`, not HTML |
| Missing images / decode errors | Run `npm run assets`; confirm files under `assets/` |
| Port in use | Stop other `node` processes or change `PORT` in `server/index.js` |
| Wrong Node version | Use Node 20+; with nvm: `nvm use` in this folder |
